《qt5开发及实例 第4版》第二章-模板库、工具类及控件

字符串类

优点：保存16位Unicode值；丰富的操作、查询、转换函数；隐式共享（写时拷贝）；内存分配高效。

操作字符串

  • “+”可以组合两个字符串，“+=”把一个字符串追加到另一个的末尾。

  • QObject::tr()函数用来使字符串便于利用插件翻译，如果程序员想在开发过程中对所有显示给用户的字符串要求必须使用tr()函数，就可以通过QT_CAST_FROM_ASCII宏屏蔽QString构造函数。

  • **QString::append()**函数用途与“+=”相同。

  • **QString::sprintf()**函数用来组合字符串，其支持的格式定义符与C++一样

  • QString::arg()函数也用来组合字符串，其类型安全，支持Unicode，允许使用“%n“参数改变字符顺序，其重载版本参数可以对字段宽度、数字基数或浮点数精度进行控制。

  • 其他组合字符串的方法：

    • insert()：在原字符串特定位置插入另一字符串；
    • prepend()：在原字符串开头插入另一字符串；
    • replace()：用指定字符串代替原字符串某些字符。

  • 去除字符串两端空白，如回车字符“\n”、换行字符”\r“、制表符”\t”、空格字符” ”等：

    • QString::trimmed()：移除字符串两端空白字符；
    • QString::simplified()：移除字符串两端空白字符，并使用单个空格字符” “代替字符串中的空白字符。

查询字符串

  • QString::startsWith()判断一个字符串A是否以某个字符串B开头，第一个参数指定B，第二个参数指定是否区分大小写（默认区分）.
  • QString::endsWith()判断一个字符串A是否以某个字符串B结尾，参数同上
  • QString::contains()判断一个一个字符串A是否包含某个字符串B，参数同上
  • 比较两个字符串：
    • **<、<=、==、>=**重载运算法按照字典序进行比较；
    • localeAwareCompare()根据本地字符集进行比较，与平台相关，一般用来向用户显示一个有序的字符串列表，当前面的字符串小于后面时，返回负整数，等于就返回0，大于就返回正整数；
    • compare()基于字符串的Unicode编码比较，返回值同上，可以设置第三个参数是否进行大小写比较。

转换字符串

  • 字符串转数值：

    • QString::toInt()转为整数类型，第一个参数是bool指针，指示转换状态，第二个参数是转换的基数，如16进制和8进制等；
    • QString::toDouble()转为双精度浮点数类型；
    • QString::toFloat()转为浮点数类型；
    • QString::toLong()转为长整类型；
    • QString::toLongLong()转为长长整数类型。

  • 字符串转换编码集，均返回QByteArray对象：

    • **toAscii()**：返回一个ASCII编码的8位字符串；
    • **toLatin1()**：返回一个Latin-1(ISO8859-1)编码的8位字符串；
    • **toUtf8()**：返回一个UTF-8编码的8位字符串；
    • **toLocal8Bit()**：返回一个本地编码的8位字符串。

  • 一个NULL字符串是QString默认构造的字符串对象，空字符串是大小为0的字符串；NULL一定是空，空不一定是NULL。

容器类

存储在容器类的元素必须可赋值，也就是一定要有：默认构造函数、拷贝构造函数、赋值操作运算符，对于QObjcet和其子类这些不能赋值的对象，可以存储指针。

QList、QLinkedList、QVector

  • QList维护了一个指针数组，各指针元素指向具体内容，因此其提供了基于下标的快速访问。

  • QList的子类有：QItemSelection、QQueue、QSignalSpy、QStringList、QTestEventList。

  • QList操作：**QList::append()、QList::prepend()、QList::Insert()**。

  • QList的两种存储策略：

    • 元素是指针类型或指针大小的类型，直接存储；
    • 只存储对象指针。

  • QLinkedList是非连续内存的链表，只能用迭代器访问，插入效率高。

  • QVector是相邻内存的数组，支持下标和迭代器访问。

  • QVector的子类有：QPloygon、QPolygonF、QStack。

  • 容器类有Java风格的迭代器和STL风格的迭代器两种：

    • Java风格的迭代器指向的是元素间隙，如第一个元素之前，两个元素之间和最后一个元素之后

    • STL风格的迭代指向的是元素

QMap、QHash

  • QHash比QMap查找速度更快。

  • QHash以任意顺序存储元素，QMap按照键的顺序存储元素。

  • QHash的键Key必须提供**operator==()和全局的qHash(Key)函数，QMap的键Key必须提供operator<()**函数。

  • 二者时间复杂度：

  • 为了实现一键多值，QMap提供**QMap<Key,T>::insertMulti()和QMap<Key,T>::values()函数，也可以使用QMultiMap<Key,T>**，其继承于QMap。

  • QHash底层维护哈希表，当存储数据的顺序无关紧要时，建议使用；**QMultiHash<Key,T>**实现一键多值。

算法与正则表达式

  • QRegExp类是正则表达式的表示类，由表达式、量词、断言组成。

  • **[AEIOU]表示匹配所有大写元音字母，[^AEIOU]表示匹配所有非元音字母，[a-z]**表示匹配所有小写英文字母。

  • 量词说明表达式出现的次数，如”x[1,2]“表示x至少有一个，至多有两个；“**+”表示至少出现一次，可以出现多次；“***”表示出现零次或多次，各量词的组成见表：

  • 断言的含义见表：

  • 实例：

    • **using(?=E\s+namespace)**表示只有using后面跟的是“ namespace”时才被匹配，\s表示匹配一个空白字符；
    • using(?!E\s+namespace)表示只有using后面跟的不是“ namespace”时才被匹配；
    • using\s+namespace表示匹配using namespace。

  • Push Button：按钮
  • Tool Button：工具按钮
  • Radio Button：单选按钮
  • Check Box：复选框
  • Command Link Button：命令链接按钮；
  • Dialog Button Box：对话框按钮

    对应QDateTime类，可以通过**QDateTime::currentDateTime()获取本地系统的时间和日期，通过date()和time()**返回其中的日期和时间

  • Horizontal Scoll Bar：横向滚动条

  • Vertical Scroll Bar：垂直滚动条

  • Horizontal Slider：横向滑块

  • Vertical Slider：垂直滑块

  • Key Sequence Edit：按键序列编辑框

  • QTimer类为定时器，可以连接定时器的**timeout()**信号于槽函数，定时时间到了就会触发槽完成任务：

    开启定时器的两种方式：

    • starttime(int time)：表示每隔time就重启定时器，重复触发timeout，可以用stop()关闭；

    • **setSingleShot(true)**：仅启动定时器一次。

项目视图组

  • List View：清单视图

  • Tree View：树形视图

    与Tabel Widget的区别：

  • Table View：表视图

  • Column View：列视图

  • Qt5使用模型视图框架完成数据和表现得分离，被称为InterView框架

    • 模型：所有模型都基于QAbstractItemModel类（抽象基类），如QStandardItemModel（用最简单的Grid方式显示Model）、QDirModel（以树形方式显示某个目录下所有子目录及其相关信息）、QStringListModel、QProxyModel（用于将旧的Model类型过度到新的类型上）；
    • 视图：所有视图都基于QAbstractItemView类（抽象基类），如QColumnView、QHeaderView、QListView、QTableView、QTreeView。

  • 视图与模型绑定时，模型必须用new创建，否则视图不能随着模型的改变而改变。
