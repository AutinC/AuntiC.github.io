小林图解网络学习笔记

1. TCP/IP 网络模型有哪几层？

1.1 应用层

专注于为用户提供应用功能，HTTP、FTP、Telnet、DNS、SMTP等；工作在操作系统中的用户态，传输层及以下则工作在内核态；用端口将应用区分开来，80 端口通常是 Web 服务器用的，22 端口通常是远程登录服务器用的；浏览器中的每个标签栏都是一个独立的进程，操作系统会为这些进程分配临时的端口号。

1.2 传输层

只需要服务好应用即可，不管具体网络传输过程；有两个传输协议，分别是 TCP 和 UDP；当传输层的数据包大小超过 MSS（除去 IP 和 TCP 头部之后，一个网络包所能容纳的 TCP 数据的最大长度） ，就要将数据包分块，方便丢失和损坏时重传单个块，每个分块称为一个 TCP 段（TCP Segment*）。

1.3 网络层

最常使用的是 IP 协议（Internet Protocol），IP 协议会将传输层的报文作为数据部分，再加上 IP 包头组装成 IP 报文，如果 IP 报文大小超过 MTU（一个网络包的最大长度，以太网中一般为 1500 字节）就会再次进行分片；用 IP 地址给设备进行编号，IPv4 协议， IP 地址共 32 位，分成了四段（比如，192.168.100.1），每段是 8 位。

知道子网掩码，该怎么计算出网络地址和主机地址呢？

  • 子网掩码与IP地址按位与运算得到网络号
  • 子网掩码取反后与IP地址进行进行按位与运算得到主机号

IP协议两个重要能力：寻址和路由；寻址作用是告诉我们去往下一个目的地该朝哪个方向走，路由则是根据下一个目的地选择路径。寻址更像在导航，路由更像在操作方向盘。

1.4 网络接口层

在 IP 头部的前面加上 MAC 头部，并封装成数据帧（Data frame）发送到网络上；作在网卡这个层次，使用 MAC 地址来标识网络上的设备；通过 ARP 协议获取对方的 MAC 地址。

2. 键入网址到网页显示，期间发生了什么？

  • 浏览器解析 URL

    URL实际上是请求服务器里的文件资源；没有路径名时，就代表访问根目录下事先设置的默认文件，也就是 /index.html 或者 /default.html；

  • 浏览器生成HTTP请求信息

  • 查询服务器域名对应的 IP 地址

    DNS服务器保存了 Web 服务器域名与 IP 的对应关系；域名中，越靠右的位置表示其层级越高；. 根域是在最顶层，它的下一层就是 .com 顶级域，再下面是 server.com权威域；根域名服务器是最高层次的，它不直接用于域名解析，但能指明一条道路；解析域名先看浏览器缓存，再看OS缓存，再看hosts文件，最后才去问本地DNS服务器。

  • 操作系统协议栈包装信息

    浏览器通过调用 Socket 库，来委托协议栈工作；协议栈的上半部分有两块，分别是负责收发数据的 TCP 和 UDP 协议，下面一半是用 IP 协议控制网络包收发操作。

    可靠传输TCP：HTTP 是基于 TCP；TCP头格式：源端口号+目标端口号+序号（解决包乱序的问题）+确认号（解决丢包的问题）+首部长度+保留+状态位（维护连接的状态）+窗口大小（流量控制、拥塞控制）+校验和+紧急指针+选项；三次握手目的是保证双方都有发送和接收的能力；在 Linux 可以通过 netstat -napt 命令查看TCP 的连接状态；

    **远程定位IP:**头格式：版本+首部长度+服务类型+总长度+标识+标志+片位移+协议+TTL+首部校验和+源地址 IP + 目标地址 IP+选项；当客户端有多个网卡即有多个IP，选哪个作为源地址呢？用路由表规则，将目标地址和表中每个条目的子网掩码与运算，找到与Destination相同的那个条目的IP作为源地址；

    两点传输MAC：头格式：发送方 MAC 地址+接收方目标 MAC 地址+协议类型；ARP广播IP获得MAC，或用ARP缓存；MAC 头部的作用就是在以太网内进行两个设备之间的包传输

  • 网卡发送

    将数字信息转换为电信号，网卡本身具有 MAC 地址，控制网卡要靠网卡驱动程序；网卡驱动获取网络包之后，会将其复制到网卡内的缓存区中，在其开头加上报头和起始帧分界符，在末尾加上用于检测错误的帧校验序列(FCS)；

  • 交换机转发

    将电信号转换为数字信号，基于以太网设计的，俗称二层网络设备，端口不具有 MAC 地址，据 MAC 地址表查找 MAC 地址，然后将信号发送到相应的端口，查不到就发给所有端口；

  • 路由器转发

    将电信号转成数字信号，基于 IP 设计的，俗称三层网络设备，各个端口都具有 MAC 地址和 IP 地址，去掉包开头的 MAC 头部，路由表查询转发目标，即根据目的IP通过ARP定MAC，在转换为电信号通过端口发出去，源 IP 和目标 IP 始终是不会变的，一直变化的是 MAC 地址

  • 服务器扒皮并发送响应报文

    TCP 头部里面还有端口号， HTTP 的服务器正在监听这个端口号，把客户端需要的网页封装在 HTTP 响应报文里，穿上 TCP、IP、MAC 头部，从网卡出去，由交换机转发到路由器，路由器就把响应数据包发到了下一个路由器，直到客户端；客户端层层扒皮后给服务器渲染生成网页。

3. Linux系统如何收发网络包？

  • 网络模型

    OSI模型

    • 应用层，负责给应用程序提供统一的接口；
    • 表示层，负责把数据转换成兼容另一个系统能识别的格式；
    • 会话层，负责建立、管理和终止表示层实体之间的通信会话；
    • 传输层，负责端到端的数据传输；
    • 网络层，负责数据的路由、转发、分片；
    • 数据链路层，负责数据的封帧和差错检测，以及 MAC 寻址；
    • 物理层，负责在物理网络中传输数据帧；

    TCP/IP 网络模型

    • 应用层，负责向用户提供一组应用程序，比如 HTTP、DNS、FTP 等;

    • 传输层，负责端到端的通信，比如 TCP、UDP 等；

    • 网络层，负责网络包的封装、分片、路由、转发，比如 IP、ICMP 等；

    • 网络接口层，负责网络包在物理网络中的传输，比如网络包的封帧、 MAC 寻址、差错检测，以及通过网卡传输网络帧等；

  • Linux网络协议栈

  • 应用程序需要通过系统调用，来跟 Socket 层进行数据交互；

  • Socket 层的下面就是传输层、网络层和网络接口层；

  • 最下面的一层，则是网卡驱动程序和硬件网卡设备；

  • Linux 接收网络包的流程

    网卡收包，通过 DMA 写进 Ring Buffer，并向 CPU 发起硬件中断，当 CPU 收到硬件中断请求后，根据中断表调用已经注册的中断处理函数。硬件中断处理函数先「暂时屏蔽中断」，告诉网卡下次再收到数据包直接写内存就可以了，再发起「软中断」；内核中的 ksoftirqd 线程专门负责软中断的处理，它从 Ring Buffer 中获取一个数据帧，交给网络协议栈进行逐层处理。

  • Linux 发送网络包的流程

    应用程序系统调用 Socket 发送数据包的接口进入内核态Socket层，内核会申请一个内核态的 sk_buff 内存，将用户待发送的数据拷贝到 sk_buff 内存，并将其加入到发送缓冲区，网络协议栈从 Socket 发送缓冲区中取出 sk_buff，并按照 TCP/IP 协议栈从上到下逐层处理，TCP拷贝一个新的 sk_buff 副本往下传递，防止误触发丢失重传，触发「软中断」告诉网卡驱动程序，驱动程序会从发送队列中读取 sk_buff，将这个 sk_buff 挂到 RingBuffer 中，接着将 sk_buff 数据映射到网卡可访问的内存 DMA 区域，最后触发真实的发送，网卡设备会触发一个硬中断来释放内存，主要是释放 sk_buff 内存和清理 RingBuffer 内存

    发送时的三次内存拷贝：

    • 用户待发送的数据 ➡内核sk_buff 内存
    • 传输层进入网络层的时候，每一个 sk_buff 都会被克隆一个新的副本出来，传送副本；
    • sk_buff 大于 MTU 时，将原来的 sk_buff 拷贝为多个小的 sk_buff。
